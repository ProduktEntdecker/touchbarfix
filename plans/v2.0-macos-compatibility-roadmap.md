# TouchBarFix v2.0 — macOS Compatibility Analysis & Prioritized Roadmap

**Date:** 2026-06-11 (post-WWDC 2026)
**Current app version:** 1.5.0

## 1. The new macOS situation

- **macOS 27 "Golden Gate"** was announced at WWDC 2026 (public beta July, release ~Sept 2026). It is **Apple Silicon only** — Intel support ends completely.
- **macOS 26 "Tahoe"** (current) is the last release that runs on any Intel Mac.
- Consequence for the Touch Bar installed base:

| Models | Chip | Max macOS |
|---|---|---|
| MacBookPro13,x / 14,x (2016–17) | Intel | Monterey 12 / Ventura 13 |
| MacBookPro15,x (2018–19) | Intel | Sonoma 14 / Sequoia 15 |
| MacBookPro16,x (2019–20) | Intel | **Tahoe 26 (final)** |
| MacBookPro17,1 (13" M1, 2020) | Apple Silicon | **Golden Gate 27+** |
| Mac14,7 (13" M2, 2022) | Apple Silicon | **Golden Gate 27+** |

- **Tahoe actively breaks the Touch Bar on M1/M2 models** (widely reported since 26.0.1): the Touch Bar disappears, Keyboard settings lose all Touch Bar references, and in some cases **no Touch Bar processes are running at all**. The common workaround is `sudo pkill TouchBarServer` (admin now often required). This is simultaneously our biggest compatibility risk and our biggest market opportunity.

## 2. Critical findings in the codebase

1. **P0 bug — Mac14,7 missing from all model lists.** The 13" M2 MacBook Pro (2022) is the *last Touch Bar Mac ever made* and one of only two models that will run macOS 27 — yet it is absent from the hardcoded lists in `TouchBarManager.swift` (`initialTouchBarDetection`, `checkTouchBarAvailability`) and `MacBookModel.swift`. Detection only survives via the process-existence fallback — which fails exactly when the Tahoe bug kills the Touch Bar processes. On the most relevant future hardware, the app can report "No Touch Bar".
2. **Bug — MacBookPro18,x wrongly listed as Touch Bar models.** The Oct 2021 14"/16" M1 Pro/Max redesign *removed* the Touch Bar. `TouchBarManager.swift` lists `MacBookPro18,3/18,4` and `MacBookModel.hasTouchBar` matches the whole `MacBookPro18` family → false positives. README's "2016–2021" claim is also wrong (correct: 2016–2022).
3. **Model list duplicated in 3 places** (twice in `TouchBarManager`, once in `MacBookModel`) — they already disagree; consolidate into `MacBookModel`.
4. **"Not running" is treated as OK.** Under the Tahoe bug, "no Touch Bar processes at all" is the *failure mode*, but `ProcessRestartStatus.notRunning` counts toward success. Needs a distinct "Touch Bar disabled by macOS" state with guided recovery.
5. **Version inconsistency:** `Info.plist` says `LSMinimumSystemVersion 11.0`, `Package.swift` targets macOS 13, README says 11.0.
6. **Deprecated APIs / UI debt:** deprecated `onChange(of:perform:)` (todo 003), plus todos 001/002/004; Tahoe's Liquid Glass redesign makes the UI look dated unless rebuilt against the macOS 26 SDK.

## 3. Prioritized proposals

### P0 — ship now (also as 1.5.1 hotfix, don't wait for 2.0)
1. **Fix model detection:** add `Mac14,7`, remove `MacBookPro18,x`, centralize the list in `MacBookModel`, update tests + README.
2. **Handle the Tahoe failure mode:** if model has a Touch Bar but no Touch Bar processes exist → new status "Touch Bar disabled by macOS" with guided fix flow (admin restart via existing AppleScript path, reboot hint, support link). Make the admin path prominent — Tahoe frequently requires root for `pkill TouchBarServer`.
3. **Build & verify on macOS 26 / 27 beta** (M1/M2 hardware), keep shipping a universal binary — the Intel installed base stays on ≤26 and must keep working. Align min-version metadata (recommend macOS 12 floor, matching 2016/17 models).

### P1 — the v2.0 core
4. **Menu-bar mode + watchdog (headline feature):** live in the menu bar, detect a frozen/vanished Touch Bar automatically and offer (or auto-apply) the fix. Turns a reactive one-click tool into "set and forget" — the justification for a 2.0.
5. **OS-aware diagnostics & messaging:** detect macOS version + chip and tailor guidance (Intel-on-Tahoe vs. M1/M2-on-Golden-Gate behave differently).
6. **UI modernization:** macOS 26 SDK / Liquid Glass pass, fix deprecated `onChange`, clear todos 001–004.
7. **Sparkle auto-update:** OS point releases (26.0.1!) change Touch Bar behavior overnight; we need to ship fixes fast to non-technical users.

### P2 — nice to have
8. **German localization** (large share of the user base), website/FAQ update for the Tahoe bug ("2016–2022 models", "macOS 26/27 ready").
9. **Opt-in anonymized stats** on macOS version vs. fix success rate to catch the next OS breakage early.

### Explicitly not recommended
- Dropping Intel support (majority of Touch Bar devices are Intel, frozen on ≤26 for years to come).
- Raising the deployment target to macOS 26.

## 4. Suggested release plan

- **v1.5.1 (now):** P0 items 1–3.
- **v2.0 (target: before macOS 27 GM, ~Sept 2026):** P1 items, marketed around "Tahoe/Golden Gate ready + automatic protection".
- **v2.x:** P2 items.
